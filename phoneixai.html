Project Phoenix: The Internal AI Assistant

An Aeitch Solution case study on developing a real-time, biometrically-authenticated assistant with a strategic RAG architecture.

The 4-Fold Challenge

Aeitch Solution needed to centralize dynamic internal information, but faced four critical hurdles to creating a truly useful and secure tool.

1. Policy & Operations

Answering nuanced questions from unstructured data like employee handbooks.

2. Dynamic Services

Managing real-time, changing data like daily meal subscriptions and menus.

3. Secure Access

Ensuring internal-only access with a seamless, high-speed biometric (facial) login.

4. Real-Time Experience

Overcoming high LLM latency to provide a fluid, conversational, and interruptible interface.

Key Innovation 1: The Strategic Pivot to RAG

An initial Knowledge Graph (KG) approach failed due to the unstructured, prose-heavy nature of our data. We strategically pivoted to a Retrieval-Augmented Generation (RAG) model, which proved superior.

Why Knowledge Graphs Failed

  • Schema Rigidity: Impossible to define a rigid schema for unstructured employee handbooks.
  • Brittle Extraction: Parsing prose into correct "node" and "edge" relationships was highly error-prone.
  • Query Failures: Natural language queries often failed to translate into formal graph queries (e.g., Cypher), returning zero results.
  • Poor Nuance: Struggled with interpretive questions ("What is the company culture like?").

Why RAG Succeeded

  • Semantic Flexibility: Easily searches raw text, retrieving entire relevant passages for the LLM.
  • Innate Simplicity: Semantic similarity search is highly robust against varied user phrasing.
  • Contextual Accuracy: Delivers comprehensive, nuanced answers perfect for complex policy questions.
  • Easy Maintenance: The knowledge base is updated simply by re-embedding new documents.

Key Innovation 2: Optimizing for Real-Time Speed

Latency was the biggest threat to user experience. Rigorous testing revealed that the initial model choice, `llama3:70b`, had an unacceptable 45-second latency. We re-tooled and selected `gemma3:27b-it-qat` for its optimal balance of speed and accuracy.

LLM Latency Comparison

The `gemma` model provided a 90%+ reduction in response latency, making a real-time conversational flow possible.

Key Innovation 3: Secure & Engaging UX

The solution required an interface that was as intelligent and secure as the backend logic, built without heavy libraries.

Biometric Auth (FAISS + Caching)

Facial embeddings are computationally expensive. We pre-cached all employee embeddings into a `.pkl` file. This, combined with FAISS for high-speed search, reduced authentication time from seconds to milliseconds.

State-Based HTML Avatar

The avatar is a composite of HTML `div`s. A JavaScript state machine adds/removes CSS classes (`is-talking`, `is-thinking`) to trigger keyframe animations, including a lip-sync synchronized with the Web Speech API.
